### 3. DB: `leads` Tablosuna `company_id` Kolonu Eklendi
**Sunucuda çalıştırıldı:**
```sql
ALTER TABLE public.leads
  ADD COLUMN IF NOT EXISTS company_id UUID REFERENCES public.companies(id) ON DELETE SET NULL;

CREATE INDEX IF NOT EXISTS idx_leads_company_id ON public.leads(company_id);
```
- Leads tablosunda `company_id` hiç yoktu — bu yüzden manuel import edilen leads RLS tarafından bloklanıyordu

### 4. RLS Politikaları Güncellendi (`leads` tablosu)
**Sunucuda çalıştırıldı:**

| Politika | Eski | Yeni |
|----------|------|------|
| SELECT | `leads_select_own_distributions` (sadece lead_distributions üzerinden) | `leads_select_company_or_distribution` (distributions VEYA company_id üzerinden) |
| UPDATE | `leads_update_admin_only` (sadece admin) | `leads_update_company_or_admin` (firma kendi leadlerini güncelleyebilir) |
| DELETE | Yoktu | `leads_delete_company_or_admin` (firma kendi leadlerini silebilir) |

**Migration dosyası:** `supabase/migrations/20260515100000_fix_leads_rls_for_manual_import.sql`

## Mevcut Çalışan Akış

```
Yeni Anfrage (buton) / Anfragen sayfasında "Neue Anfrage"
    ↓
ManualImport sayfası → metin yapıştır → AI ile ekstrakt
    ↓
Form düzenle → "Anfrage speichern" butonu
    ↓
Edge function: import-manual-lead
  → leads tablosuna company_id ile kaydet
    ↓
/firma/anfragen sayfasına yönlendir
    ↓
Anfragen listesinde kart görünür
  → "Offerte erstellen" → /firma/offerten/neu?lead=<id>
  → "V. Besichtigung" → /firma/besichtigungen
  → "Details" → dialog
  → (sil) → leads tablosundan sil
```

## Bilinen Kalan Sorunlar / Yapılabilecekler

1. **"V. Besichtigung" butonu** — Besichtigungen sayfasına gidiyor ama lead bilgisini otomatik doldurmuyor. İleride `lead_id` parametresini Besichtigungen sayfasında işleyip "Neue Besichtigung" formunu öndoldurmak gerekebilir.

2. **Eski leads (company_id'siz)** — Migration öncesi import edilen leadlerin `company_id` kolonları NULL. Bu leadler RLS nedeniyle listede görünmeyebilir (admin değilse). Gerekirse:
   ```sql
   -- Firm sahibinin company_id'sini bul ve NULL olan leadleri güncelle
   UPDATE public.leads SET company_id = '<company_uuid>' WHERE company_id IS NULL;
   ```

3. **leads tablosunun NOT NULL kısıtları** — `customer_first_name`, `customer_last_name`, `customer_email`, `customer_phone`, `from_plz`, `from_city` NOT NULL olarak tanımlı. Edge function bunları boşsa "Unbekannt"/"" ile dolduruyor. İleride bu alanları NULL kabul edecek şekilde gevşetilebilir.
